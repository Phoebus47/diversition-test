# Contributing

- Follow [docs/CODING_STANDARDS.md](docs/CODING_STANDARDS.md) (lint, format, types, no hardcoding, file size).
- Run `npm run lint` and `npm run test:coverage` before committing; ensure Quality Gate passes if you run `npm run sonar`.
- Use conventional commits: `type(scope): subject` (see [.cursor/rules/commit-format.mdc](.cursor/rules/commit-format.mdc) or Commitlint config).
